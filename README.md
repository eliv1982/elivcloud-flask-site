# ElivCloud — Flask portfolio MVP

Первая версия реального личного сайта-портфолио `ElivCloud` в нише:
- prompt engineering;
- AI automation;
- RAG-ассистенты;
- AI-боты и workflows для текстовых процессов.

Проект подготовлен как MVP личного сайта и одновременно закрывает требования учебного задания по структуре Flask-приложения.

## О проекте

Сайт сделан как полноценное Flask-приложение с шаблонами, страницей кейсов, контактной формой и простой админкой для просмотра заявок.

На публичных страницах сайт оформлен как профессиональный личный бренд:
- без упоминаний учебного статуса;
- с чётким позиционированием и специализацией;
- с кейсами и контактным сценарием для потенциальных клиентов.

## Ниша и специализация

ElivCloud помогает бизнесу и экспертам:
- автоматизировать повторяющиеся текстовые процессы;
- внедрять AI-ботов и RAG-ассистентов;
- структурировать обращения, отзывы и заявки;
- строить AI-workflows с интеграциями.

## Польза для клиента

- Экономия времени на повторяющихся коммуникациях;
- прозрачная обработка заявок и обратной связи;
- быстрый запуск пилотного прототипа;
- ответы ассистента на основе базы знаний (RAG);
- понятный пошаговый процесс внедрения.

## Кейсы в проекте

1. `review-analysis` — AI-анализ отзывов клиентов  
2. `weather-teller` — Telegram-бот с AI-пояснениями погоды  
3. `rag-assistant` — RAG-ассистент по базе знаний и документам  
4. `kitchen-helper` — Kitchen Helper AI Assistant  
5. `moodmuse` — MoodMuse: AI-генерация открыток

## Быстрый запуск локально

### 1) Создать виртуальное окружение

Windows (PowerShell):

```bash
python -m venv .venv
.venv\Scripts\Activate.ps1
```

macOS/Linux:

```bash
python -m venv .venv
source .venv/bin/activate
```

### 2) Установить зависимости

```bash
pip install -r requirements.txt
```

### 3) Подготовить переменные окружения

Скопируйте пример:

```bash
copy .env.example .env
```

или создайте `.env` вручную на основе `.env.example`.

### 4) Запустить приложение

```bash
python app.py
```

Приложение будет доступно по адресу: [http://127.0.0.1:5000](http://127.0.0.1:5000)

- Главная страница: [http://127.0.0.1:5000/](http://127.0.0.1:5000/)
- Кейсы: [http://127.0.0.1:5000/cases](http://127.0.0.1:5000/cases)
- Контакты: [http://127.0.0.1:5000/contact](http://127.0.0.1:5000/contact)
- Админка: [http://127.0.0.1:5000/admin/login](http://127.0.0.1:5000/admin/login)

Тестовые данные для локального запуска:
- `ADMIN_USERNAME=admin`
- `ADMIN_PASSWORD=admin123`

## Контактная форма и админка

### Форма `/contact`
- поля: name, email, phone, subject, message;
- валидация через Flask-WTF;
- CSRF protection включена;
- после отправки заявка сохраняется в SQLite;
- пользователь получает flash-уведомление.

### Админ-панель
- вход: `/admin/login`;
- список заявок: `/admin/messages`;
- отметить прочитанной: `/admin/messages/<id>/read`;
- удалить заявку: `/admin/messages/<id>/delete`;
- выход: `/admin/logout`.

## Подготовка к деплою

Для запуска в production предусмотрен `gunicorn`:

```bash
gunicorn app:app
```

Перед публичным деплоем обязательно:
1. Заменить `SECRET_KEY` на уникальный и безопасный.
2. Заменить `ADMIN_PASSWORD` на сложный пароль.
3. Хранить секреты только в переменных окружения (не в репозитории).
4. Проверить настройки базы данных и логирования под окружение хостинга.

## Что реализовано по заданию

- Flask + Jinja2 templates;
- Bootstrap 5 (CDN) + адаптивный дизайн;
- SQLAlchemy + SQLite (`instance/site.db`);
- Flask-WTF формы и CSRF;
- Flask-Login авторизация админа;
- кейсы (`/cases`, `/cases/<slug>`);
- контактная форма с сохранением заявок;
- админка просмотра/статуса/удаления заявок;
- логирование ключевых действий;
- запуск локально через `python app.py`;
- готовность к запуску через `gunicorn`.
# ElivCloud — static-first portfolio

Учебный одностраничный сайт для личного бренда в сфере prompt engineering и AI automation.  
Проект сделан как реальная база портфолио, которую можно развивать дальше и публиковать на домене `elivcloud.org`.

## Что это за проект

`ElivCloud` — это landing page, который показывает:
- специализацию;
- практическую пользу для клиента;
- портфолио/демонстрационные кейсы;
- понятный процесс работы;
- контакты для связи.

Сайт полностью статический (`HTML + CSS + JS`) и работает без backend.

## Почему выбран static-first подход вместо Flask

Для задачи личного бренда важнее быстрый запуск, простая поддержка и лёгкий деплой, чем серверная логика.  
Статический подход позволяет:
- публиковать сайт на GitHub Pages / Cloudflare Pages / Netlify;
- не обслуживать backend и базу данных;
- минимизировать риски и стоимость поддержки;
- быстрее обновлять контент портфолио.

## Как сайт закрывает требования урока

Реализованы все ключевые блоки:
- Hero-блок с позиционированием и CTA-кнопками;
- блок специализации;
- блок пользы для клиента;
- блок направлений “Чем помогаю”;
- блок кейсов;
- блок процесса работы;
- контактный блок и форма (без отправки на сервер);
- адаптивная вёрстка для desktop/mobile.

## Как открыть локально

1. Склонируйте или скачайте проект.
2. Откройте папку проекта в браузере через `index.html`.
3. Для корректной разработки удобно использовать Live Server (VS Code/Cursor) или любой локальный static server.

Пример через Python:

```bash
python -m http.server 8000
```

После запуска откройте: [http://localhost:8000](http://localhost:8000).

## Как задеплоить позже

### GitHub Pages
1. Загрузить проект в GitHub-репозиторий.
2. Включить Pages в настройках репозитория (`Deploy from branch`, ветка `main`, папка `/root`).
3. Получить публичный URL и при необходимости привязать домен `elivcloud.org`.

### Cloudflare Pages
1. Подключить репозиторий к Cloudflare Pages.
2. Указать build command: пусто.
3. Указать output directory: `/` (корень проекта).
4. Привязать кастомный домен `elivcloud.org` через DNS.

### Netlify
1. Импортировать репозиторий в Netlify.
2. Build command: пусто.
3. Publish directory: `.` (текущая папка).
4. Настроить домен `elivcloud.org`.

## Что реализовано технически

- современный визуальный стиль (тёмный технологичный фон + мягкий голубой акцент);
- адаптивная сетка и mobile-навигация;
- hover-эффекты карточек и кнопок;
- плавная прокрутка по якорям;
- читаемая типографика;
- JS-обработчик формы с уведомлением для учебной версии.

## Обоснование архитектуры

В уроке Flask использовался как учебный пример для демонстрации backend-структуры, шаблонов, формы и простой админки. Для личного сайта ElivCloud выбран static-first подход, потому что задача личного бренда — быстро и понятно представить специализацию, пользу и кейсы, а также иметь возможность разместить сайт на собственном домене без поддержки серверной части. Такой вариант проще деплоить, легче поддерживать и безопаснее для публичного портфолио. По содержанию сайт сохраняет структуру урока: главный экран, описание специализации, блок пользы для клиента, кейсы и контактный блок.
